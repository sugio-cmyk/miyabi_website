# ブログ記事テンプレート

## 記事構成（SEO・LLMO最適化）

```
① タイトル（h1）- WordPressが自動生成

② リード文（段落）
   - 読者の悩み + この記事で解決できること
   - キーワードを自然に含む

③ 目次 - VK Blocks自動挿入（最初のh2の前）

④ この記事のポイント（Border Box）
   - 3つ程度の要点

⑤ 本文セクション（h2 × 2〜4個）
   - 必要に応じてh3で細分化
   - 注意点はAlert使用
   - Q&AはFAQ使用

⑥ まとめ（Border Box）または CTA
```

---

## 見出し階層のルール（重要）

目次の正しい表示のため、見出しは**必ず階層順**に使用すること。

```
h2 → h3 → h4 の順番で使う（スキップ禁止）
```

| レベル | 用途 | 例 |
|--------|------|-----|
| **h2** | 大セクション | 「社会保険の届出期限」「よくある質問」 |
| **h3** | 中セクション（h2の下位） | 「届出に必要な書類」「届出の流れ」 |
| **h4** | 小セクション（h3の下位） | 「必要な情報①」「必要な情報②」 |

### ❌ ダメな例
```
h2 社会保険の届出期限
h4 届出に必要な書類  ← h3を飛ばしている
```

### ✅ 良い例
```
h2 社会保険の届出期限
h3 届出に必要な書類
h2 雇用保険の届出期限
h3 届出に必要な書類
```

## 対応ブロックタイプ

| type | 用途 | VK Blocks |
|------|------|-----------|
| `heading` | 見出し（h2-h4） | - |
| `paragraph` | 段落 | - |
| `list` | リスト | - |
| `warning` | 注意ボックス | Alert |
| `faq` | Q&A | FAQ2 |
| `box` | 汎用ボックス | Border Box |
| `table` | テーブル | - |
| `spacer` | スペーサー | - |

---

## VK Blocks設定

- **目次の位置**: 「最初の見出しの前」に設定
- **投稿タイプ**: 投稿のみ有効

---

## 使い方

```bash
py tools/block_generator.py input.json -o output.txt
```
